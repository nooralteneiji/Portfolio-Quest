# Portfolio Quest League (PQL)

A paper-based operating system for your job. The system is built to:

- Track only portfolio-grade work (no admin noise in your main score)
- Prevent "stretching tasks longer than they should"
- Create steady pace through weekly finish lines
- Keep you unblocked via an Ops lane
- Maintain continuity through capture → processing → review

---

## 1. What the Game Optimizes For

### The game's definition of progress

Progress is not time spent. Progress is **artifacts shipped**.

If an output exists in a form that can be reviewed/shared, the game counts it.

### The game's core design principle

You run three lanes in parallel, but only one lane counts toward your portfolio pace:

| Lane | Purpose | Measured by |
|------|---------|-------------|
| **Portfolio Lane** (Quest Lane) | Tangible deliverables | XP and ship days |
| **Ops/Admin Lane** (Bottleneck Lane) | Admin and follow-ups | Tokens + waiting-for discipline |
| **Mastery Lane** (Skill Lane) | Learning and development | Sessions/lessons (not XP) |

This separation is what keeps the system clean.

## 2. Glossary and Hierarchy

### Epic
A portfolio project big enough to deserve space in your year-end achievements. Epics contain milestones and artifacts.

### Milestone
A phase within an Epic (design → draft → revise → pilot → results). Milestones help you structure what "done" looks like.

### Artifact
A tangible output that exists as a doc/deck/table/script/memo. Artifacts earn XP when shipped.

### Slice
A deliberately small artifact created to prevent stalling when you're thinking/reading but nothing exists yet. Slices count as artifacts (small XP), and are your anti-procrastination tool.

### Quest
A weekly commitment tied to one Epic and one Friday Finish Line artifact. You run max 2 Quests/week.

### Move
A weekly "play": a work package that produces one or more artifacts. You choose exactly 3 Moves per week for portfolio work. Moves are the only portfolio work you allow yourself to advance that week.

### Ship
A Ship happens when an artifact exists (drafts count). You don't "ship" an Epic or a Quest; you ship artifacts.

### Ops Token
A 15-minute unit used to schedule admin work so it doesn't float or get ignored.

### Waiting For + Ping Date
A record of a dependency outside your control plus the next date you will follow up.

### Hard Date
A meeting/deadline that must not be forgotten. Hard dates get duplicated into a dedicated list immediately.

### Parking
A deliberate "not this week" status to protect focus and reduce cognitive load.

## 3. Paper Assets You Maintain

You can run this entirely in an A6 notebook + index cards.

### A) Epic Quest Cards (index cards)

One card per Epic. Each card contains:

- Epic name + purpose
- Definition of done (portfolio deliverable)
- Milestones
- Artifact list + XP values
- Stakeholders / where feedback comes from

These are your "project library."

### B) Weekly Command Card (A6 front/back)

Your weekly control panel includes:

- 2 Quests + Friday finish line for each
- 3 Moves (for the week)
- Planned XP + baseline thresholds
- Daily ship tracker (Mon–Fri)
- Ops tokens per day
- Waiting For list + ping schedule
- Mastery trackers
- Parking lot + hard dates
- Friday Boss Fight: what shipped + who saw it + feedback

### C) Daily A6 Page (one page per day)

Contains:

- Today's hard dates
- Plan: one portfolio focus (from your Moves), ops blocks, mastery
- A big INBOX capture area
- End-of-day processing marks

### D) Permanent Lists in the Notebook

- **Hard Dates** (next 30–45 days)
- **Waiting For** (dependencies + ping dates)
- **Parking Lot** (future ideas and non-this-week tasks)

## 4. The Rules of the Game

### Rule 1: Portfolio Filter
Only track (and score) work that you'd be proud to cite in your year-end portfolio. If it's admin, follow-up, logistics, scheduling, tool setup: it goes to Ops lane and earns no portfolio XP.

### Rule 2: Max 2 Quests Per Week
Each Quest has:
- One Epic
- One Friday Finish Line artifact
- A clear "done" statement

### Rule 3: Exactly 3 Moves Per Week (portfolio lane)
Moves define what you're allowed to advance this week. If something new appears mid-week and must be done, it must replace a Move (swap rule).

### Rule 4: Ships Are Artifacts (including slices)
A day counts as a "Ship Day" if you created at least one artifact/slice.

### Rule 5: Slice Rule (anti-stall)
If you've spent multiple work sessions "thinking/reading" with no artifact, you must create a Slice immediately. A Slice is the smallest visible version of the work:
- A one-page framework
- A synthesis table
- An outline with headings + bullets
- A draft index/schema
- A decision note

### Rule 6: Ops Must Be Scheduled (tokens)
Admin tasks don't float. They live in ops tokens on the weekly card and ops blocks on the daily page.

### Rule 7: Dependencies Must Live in "Waiting For"
If you're waiting on someone: move the item to Waiting For, assign a next ping date, and only follow up when the ping date arrives.

### Rule 8: Hard Dates Get Doubled Immediately
If something has a date/time: capture it in your daily inbox AND immediately copy it to Hard Dates.

### Rule 9: Mastery Is Separate
Reading/courses are tracked for consistency but don't affect portfolio pacing unless they generate a portfolio artifact.

## 5. Scoring and Pace

### XP is attached to artifacts only
Each artifact has a fixed XP value on the Epic card (normalized by your chosen XP table).

### Weekly baseline (rolling)
You track pace by comparing each week's earned XP to your rolling baseline:

| Metric | Formula |
|--------|---------|
| Baseline XP | Median of the last 3 weeks |
| On Track | ≥ 80% of baseline |
| Ahead | ≥ 120% of baseline |

Lagging / On Track / Ahead is determined by your thresholds plus the ship-day rule.

### Ship-day rule
Your week is considered healthy only if ship days are frequent enough (e.g., 3+ days/week). This stops "one Friday dump" from hiding slow drift.

### CEO Leverage (executive impact metric)
Every shipped artifact can be tagged with a **leverage type** that captures how it multiplied the CEO's effectiveness. This shifts the narrative from "I completed tasks" to "I made the CEO's job easier."

| Leverage Type | Code | Meaning |
|---------------|------|---------|
| **Decision Enabled** | DECIDE | Prepared a brief, analysis, or recommendation so the CEO decides faster |
| **Problem Absorbed** | ABSORB | Handled something that would have escalated to the CEO |
| **Alignment Created** | ALIGN | Got teams or stakeholders aligned without CEO intervention |
| **Capacity Created** | CLEAR | Freed CEO time by automating, delegating, or streamlining |

The HUD displays a **LEVERAGE** counter showing this week's breakdown (D/A/L/C). The weekly status and wrapup views include a **CEO LEVERAGE** brief that summarizes your organizational impact — what the CEO would hear if asked "What is this person doing for me?"

Leverage is tracked per-artifact in the XP ledger alongside XP, and aggregated weekly.

## 6. Weekly Workflow (the game loop)

### Step 1: Weekly Reset (start of week)

On Monday morning (or Sunday evening), do this in order:

**Close last week**
- Review last week's "open" artifacts
- Mark what shipped
- Record XP earned
- Note what got stuck and why (1 sentence)

**Review dependencies**
- Scan Waiting For and pings due
- Update ping dates as needed

**Select your Quests (2 max)**
- Choose 1–2 Epics to actively advance
- For each Quest, define the Friday Finish Line artifact (one artifact each)

**Select your 3 Moves**
- Move 1 produces Quest 1 finish line
- Move 2 produces Quest 2 finish line
- Move 3 is the most critical prerequisite or highest-risk blocker to protect finish lines

**Declare committed vs stretch**
- Committed artifacts are what you'll deliver no matter what
- Stretch artifacts are only attempted after committed work is shipped

**Update Parking**
- Explicitly park anything not in play this week
- Parking reduces mental clutter and prevents stealth scope creep

**Set Ops tokens per day**
- Allocate tokens per day (e.g., 2/day + 4 on Friday)
- Write pings due by day

**Set Mastery targets**
- Choose weekly mastery goal (sessions/lessons)

Your weekly card now has everything you need.

## 7. Daily Workflow

### Morning (2–5 minutes)
- Check Hard Dates and copy today's meetings/deadlines onto today's page
- Choose which Move you are playing today (only one primary portfolio focus)
- Allocate Ops blocks (email/docs batch + calls/follow-ups batch)
- If you want: commit to one mastery session today

### During the Day (capture)
Everything unexpected goes into the INBOX on the daily page. Do not decide where it goes in the moment, except: if it includes a hard date → copy to Hard Dates immediately.

### End of Day (wrap-up, 6–10 minutes)

**Mark what you shipped today.**

**Process INBOX items using routing marks:**
- → Portfolio (ties to an Epic + artifact)
- → Ops (schedule tokens)
- → Waiting For (add ping date)
- → Hard Date
- → Parking
- × Drop

**Close loops:**
- Any open admin item gets a scheduled token
- Any external dependency gets a Waiting For entry + ping date

**Update your daily ship tracker box on the weekly card.**

---

## 8. Friday Boss Fight

This is a **non-negotiable** weekly accountability ritual.

1. **Ship** the Friday finish line artifacts (even if draft-quality)
2. **Get them seen** by a person (boss, stakeholder, teammate)
3. **Capture feedback** (even one bullet)

**Record:**
- What shipped
- Who saw it
- The next change or decision

**Compute:**
- Weekly XP earned
- Ship days achieved
- Status vs baseline (lagging / on track / ahead)

Boss Fight is the difference between "I worked" and "I delivered."

---

## 9. How to Handle New Work Without Breaking the Game

When something new pops up mid-week, follow this decision tree:

| Question | Action |
|----------|--------|
| Is it a hard date? | → Hard Dates (immediately) |
| Is it a dependency or blocker for others? | → Ops tokens + Waiting For (ping date) |
| Is it portfolio-grade and urgent this week? | → It must replace one of your 3 Moves (swap rule) |
| Is it portfolio-grade but not urgent? | → Parking Lot (consider next week) |
| Is it training/reading? | → Mastery (separate tracker) |

This keeps you responsive without losing focus.

## 10. How to Prevent Stretching Tasks

Stretching happens when work is fuzzy. The game prevents it with:

### A) Finish line artifacts
Your Quest is always defined by an artifact. This prevents "work on X" from dragging.

### B) Moves are phrased as "Create [artifact]"
Moves must be written as a deliverable, not a vague activity.

### C) Slice rule
When you feel yourself thinking without output, you create a slice. Slices create momentum and reduce anxiety.

## 13. Troubleshooting

### If you feel behind but you're not missing deadlines
Your finish lines may be too big or too vague. Convert the week into smaller artifacts and use slices earlier.

### If Ops is taking over
Increase tokens, but keep them in their lane. Reduce portfolio commitments rather than letting admin silently crush your Quest time.

### If you keep changing priorities mid-week
Use the swap rule: replace Moves instead of adding more. Anything else goes to Parking.

### If you're reading a lot but shipping little
Require a synthesis artifact for reading (a table, memo, or slide skeleton). Reading only counts when it produces a visible output.
